What is a UUID?

A UUID (Universally Unique Identifier) is a 128-bit number used to uniquely identify information in computer systems.

In Minecraft, UUIDs are used to identify resource packs, behavior packs, and other components to ensure each one is unique.

Why UUIDs are important: Minecraft uses UUIDs to distinguish between different resource packs. If two packs have the same UUID, Minecraft won't know which one to use.

UUID Format

A standard UUID looks like this: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

Where each 'x' is a hexadecimal digit (0-9, a-f).

How to Use UUIDs in Your manifest.json

Once you've generated your UUIDs, use them in your manifest.json file like this:

{
  "format_version": 2,
  "header": {
    "description": "Your pack description",
    "name": "Your pack name",
    "uuid": "22b87fae-158c-4663-aa29-a3d70acbf0b3",
    "version": [1, 0, 0],
    "min_engine_version": [1, 16, 0]
  },
  "modules": [
    {
      "description": "Your module description",
      "type": "resources",
      "uuid": "19c0aaaf-dfcb-4de9-9184-eb1ba17cd774",
      "version": [1, 0, 0]
    }
  ]
}

Important: You need two different UUIDs - one for the header and one for the module. They must be different from each other!

UUID Best Practices

  • Generate new UUIDs for each resource pack you create
  • Never reuse UUIDs from other packs
  • Keep a record of which UUIDs you've used for which packs
  • If you update your pack, keep the same UUIDs but change the version number

Common Mistakes

  • Using the same UUID for header and module
  • Copying UUIDs from tutorial examples
  • Not changing UUIDs when creating a new pack
  • Using invalid UUID formats
